# Updating this artifact

This project was generated by the **meta template** `cyan/new` — the official CyanPrint template that scaffolds artifacts. `cyanprint update` regenerates it from the latest version and three-way merges (prior output / your current files / new output):

- Files you never touched update cleanly.
- Files you edited that upstream did not change are kept.
- Files BOTH sides changed either merge through a resolver or land in `.cyan_conflicts/<path>.target` for manual resolution.

## Practice

- Run `cyanprint update . --template cyan/new` to pull the latest scaffolding — CI workflows, the CI script, these skills, and the vendored `@cyanprint/sdk` types. Let update own those files; do not hand-edit them, or you will create update conflicts.
- You own: `src/`, `cyan.yaml` metadata, your `package.json` dependencies, and your tests. Bump those yourself (e.g. `bun update`), then re-run `cyanprint test`.
- Your original answers and deterministic state persist in `.cyan_state.yaml`, so update reuses them without re-asking.
- After any change, run `cyanprint test` — every case must pass and validations must exit 0.
- If conflicts appear under `.cyan_conflicts/`, resolve them in place and re-run the update; a conflict-free update clears the folder.
